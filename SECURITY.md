# Security Policy

SGAD is a workflow and governance framework. Security-sensitive changes should be treated as R3.

Examples:

- authentication or authorization
- RBAC and tenant isolation
- payment, billing, or deletion
- secret handling
- external side effects
- compliance workflows

For R3 changes:

- require human approval
- document rollback
- update the risk register
- link tests in the evidence matrix
- avoid autonomous rollout

Please report security issues privately to the repository owner rather than opening a public issue.
